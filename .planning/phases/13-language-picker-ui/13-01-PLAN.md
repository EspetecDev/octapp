```svelte
<script lang="ts">
  import { locale, setLocale, type SupportedLocale } from '$lib/i18n/locale.svelte.ts';

  const LOCALES: { code: SupportedLocale; label: string }[] = [
    { code: 'en', label: 'English' },
    { code: 'ca', label: 'Català' },
    { code: 'es', label: 'Español' },
  ];
</script>

<div class="flex gap-2" role="group" aria-label="Language">
  {#each LOCALES as { code, label }}
    <button
      type="button"
      class="
        flex-1 rounded-full px-3 text-sm font-medium transition-all duration-150
        min-h-[44px] cursor-pointer
        {locale.current === code
          ? 'bg-text-primary text-bg border-2 border-text-primary'
          : 'bg-surface border-2 border-border text-text-secondary hover:text-text-primary'}
      "
      onclick={() => setLocale(code)}
      aria-pressed={locale.current === code}
    >
      {label}
    </button>
  {/each}
</div>
```

Key implementation rules:
- Import path MUST be `$lib/i18n/locale.svelte.ts` with the `.ts` extension (matches existing convention)
- Import `type SupportedLocale` as a type import — do NOT import from `$lib/paraglide/runtime.js` (locked decision in STATE.md)
- Active pill: `bg-text-primary text-bg` (#f9fafb fill + #0f0f0f text) — highest contrast, matches CTA button
- Inactive pill: `bg-surface border-border text-text-secondary` — matches form input style
- `aria-pressed` on each button — correct ARIA pattern for toggle/selection groups
- `type="button"` is mandatory — prevents accidental form submission if component is ever placed inside a `<form>`
- Do NOT add `justify-center` to the flex container — `flex-1` on each button makes them equal width and fills the row naturally
- Do NOT wrap in a `<label>` or `<fieldset>` — `role="group"` with `aria-label="Language"` is sufficient

Reactivity note: Reading `locale.current` in `aria-pressed={locale.current === code}` and in the class expression puts `locale.current` into Svelte 5's reactive graph. When `setLocale()` mutates `locale.current`, Svelte re-renders this component AND any ancestor that also reads `locale.current`. The join page re-renders because `+page.svelte` contains `m.*()` calls — those run again with the new locale, updating all strings.
