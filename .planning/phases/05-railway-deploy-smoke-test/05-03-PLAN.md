<task type="checkpoint:human-verify">
  <name>Task 2: WebSocket 101 upgrade verification in DevTools</name>
  <files></files>
  <read_first>.planning/research/PITFALLS.md</read_first>
  <action>Human-verify checkpoint: no automated action. Follow the how-to-verify steps below.</action>
  <what-built>
The server has a /ws WebSocket endpoint. The client (socket.ts) connects to it automatically on page load using wss:// on HTTPS pages.
  </what-built>
  <how-to-verify>
This check confirms the WebSocket is upgrading correctly — the health check cannot confirm this (Pitfall 5 from PITFALLS.md).

**Steps:**
1. Navigate to `<RAILWAY_URL>/` with DevTools open
2. In DevTools, click the **Network** tab
3. In the filter bar, click **WS** (or type `ws` in the filter box) to show only WebSocket requests
4. Hard-refresh the page (Cmd+Shift+R on Mac, Ctrl+Shift+R on Windows) to capture the initial connection
5. You should see a request to `/ws` in the list

**What to check on the /ws request:**
- **Status:** must be `101 Switching Protocols` (NOT 200, NOT 400, NOT 502)
- **Protocol:** click the /ws row → Headers tab → Request URL must start with `wss://` (not `ws://`)
- **Messages tab:** click the /ws row → Messages tab — you should see incoming PING messages every 30 seconds (shown as binary frames or `{"type":"PING",...}`)

**If status is 400:** WebSocket upgrade failed — check Railway logs for errors in the `open` handler
**If status is 502:** Railway proxy is not reaching the server — health check may be passing on a stale container
**If URL shows ws:// instead of wss://:** Mixed-content bug — check src/lib/socket.ts line 183

Record: WS status = 101 YES/NO, Protocol = wss:// YES/NO, PING messages visible = YES/NO
  </how-to-verify>
  <verify>Human confirms WS status 101 by typing "ws ok 101" as the resume signal.</verify>
  <done>DevTools Network WS tab shows /ws with status 101 Switching Protocols. Request URL starts with wss://. PING messages appear in the Messages tab every ~30 seconds.</done>
  <acceptance_criteria>
    - DevTools Network WS tab shows a row for /ws
    - Status for /ws is "101 Switching Protocols"
    - Headers tab shows Request URL starting with "wss://" (not "ws://")
    - Messages tab shows at least one incoming frame with PING content within 35 seconds
  </acceptance_criteria>
  <resume-signal>Type "ws ok" with the status code you saw (e.g., "ws ok 101"), or describe the error</resume-signal>
</task>
