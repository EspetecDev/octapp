<task type="checkpoint:human-verify" gate="blocking">
  <name>Task 2: E2E verification roundtrip — per D-08</name>
  <what-built>
Import Config button, confirm mode, error strip, form population, and restoredFromState guard — all implemented in Plan 10-01.
  </what-built>
  <how-to-verify>
Open /admin/setup in your browser (with a valid admin token). Run these 7 steps in order:

**Step 1 — Verify three-button bar:**
Confirm the sticky bar shows three buttons: Import Config | Export Config | Save Setup — each taking equal width (flex-1). Import Config must be enabled even if the form is empty.

**Step 2 — Test error strip (invalid file):**
Click Import Config → select `/tmp/octapp-invalid-test.json`.
Expected: Error strip appears above the sticky bar showing "⚠ Missing or invalid 'chapters' field — expected an array". Form is unchanged. The error persists (does not auto-dismiss). Clicking Import Config again → new file picker opens, strip clears on next selection attempt.

**Step 3 — Export current config:**
Fill in at least one valid chapter (name, scavenger clue, reward, one trivia question with correct + 3 wrong answers).
Click Save Setup to confirm server has this state.
Click Export Config → file `octapp-setup.json` downloads (or opens new tab on iOS).

**Step 4 — Reload the page:**
Hard refresh the /admin/setup page (Cmd+Shift+R or equivalent) to reset the form to server state. The form should repopulate from the server via the existing STATE_SYNC path.

**Step 5 — Import the exported file:**
Click Import Config → select `octapp-setup.json`.
Expected: Sticky bar enters confirm mode: shows "Replace setup?" label + [Cancel] [Yes, Replace] buttons.

**Step 6 — Cancel and re-import:**
Click Cancel. Expected: Normal three-button bar returns, form is unchanged.
Click Import Config again → select `octapp-setup.json` again.
Click Yes, Replace.
Expected: Form is populated with the exported chapter data. Import Config button briefly shows "Imported!". Form now contains the same chapter values as the exported file.

**Step 7 — Save and verify server state:**
Click Save Setup.
Open a second browser tab at the same /admin/setup URL (or any game page that shows server state).
Confirm the server state matches the imported config values (chapter names, questions, etc.).
