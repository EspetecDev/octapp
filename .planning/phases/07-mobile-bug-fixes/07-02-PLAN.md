<objective>
Extend server crash protection (FIX-02) by adding an `unhandledRejection` handler alongside the existing `uncaughtException` handler, and add a `/test-crash` route for manual verification.

Purpose: The existing Phase 5 `uncaughtException` handler only catches synchronous throws. Async Bun servers are equally likely to produce unhandled Promise rejections (rejected Promises with no `.catch()`). Without a handler, Node.js/Bun terminates the process — wiping all in-memory game state. This plan adds the missing handler.

Output: `server/index.ts` extended with `unhandledRejection` handler and a `/test-crash` route that triggers a rejection so the fix can be verified locally and in Railway logs.
</objective>
