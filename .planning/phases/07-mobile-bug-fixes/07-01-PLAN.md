<objective>
Fix the Android hardware back button bug (FIX-01) on both active player pages.

Purpose: During a live game session, pressing the Android back button triggers a browser navigation (popstate) that leaves the page, breaking the session for that player. This fix silently blocks all navigation away from the groom and party pages while the player is on them.

Output: Both page files modified with a `beforeNavigate` guard (blocks SvelteKit client-side navigation) and a `history.pushState` dummy entry on mount (so the hardware back button hits the dummy entry instead of navigating back to the previous page).
</objective>
