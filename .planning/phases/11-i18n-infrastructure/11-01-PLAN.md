**Step 3 — Verify `vite.config.ts`.** After scaffold, the file MUST match this exact shape:
```typescript
import { paraglideVitePlugin } from "@inlang/paraglide-js/vite";
import { sveltekit } from "@sveltejs/kit/vite";
import { defineConfig } from "vite";
import tailwindcss from "@tailwindcss/vite";

export default defineConfig({
  plugins: [
    tailwindcss(),
    sveltekit(),
    paraglideVitePlugin({
      project: "./project.inlang",
      outdir: "./src/lib/paraglide",
      strategy: ["localStorage", "preferredLanguage", "baseLocale"],
    }),
  ],
});
```
Plugin order is mandatory: `tailwindcss()` and `sveltekit()` BEFORE `paraglideVitePlugin()`. The `strategy` array MUST NOT contain `"url"`. If the scaffold generated different strategy options or plugin order, rewrite the file to match this shape exactly.

**Step 4 — Add `paths: { relative: false }` to `svelte.config.js`.** The scaffold does not add this. Manually add it inside the `kit` config object (per D-05/PITFALL-5 — prevents sub-route 404s with adapter-static when Paraglide is present):
```javascript

    pages: "build",
    assets: "build",
    fallback: "index.html",
    precompress: false,
    strict: false,
  }),
  paths: { relative: false },   // Required for Paraglide + adapter-static
},
```

**Step 5 — Verify `project.inlang/settings.json`.** The scaffold generates this. Confirm it matches (correct if different):

`sourceLanguageTag: "en"` is the compile-time fallback — missing keys in ca/es render the English string with no runtime error.
