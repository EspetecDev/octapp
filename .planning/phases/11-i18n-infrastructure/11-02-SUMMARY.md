## Decisions Made

- **Class-based $state pattern** for exported mutable state: Svelte 5 enforces `state_invalid_export` — you cannot export a `$state` variable that is also reassigned within the same module. The fix is a class (`class LocaleState { current = $state() }`) with an exported instance. Components access `locale.current`.
- **No `onSetLocale` bridge** — this Paraglide v2.16.0 runtime does not export `onSetLocale` or `onSetLanguageTag`. The research/plan expected one of these; neither exists. Instead, `locale.current` is updated manually after every `paraglideSetLocale()` call with `{ reload: false }`.
- **Dual localStorage keys** — `octapp:locale` is our app's key (for the FOUC script and persistence); `PARAGLIDE_LOCALE` is Paraglide's internal key. Both exist simultaneously.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Svelte 5 state_invalid_export: class-based $state pattern required**
- **Found during:** Task 1 build verification
- **Issue:** `vite build` failed with `state_invalid_export` — Svelte 5 does not allow exporting a `$state` variable that is reassigned within the same module. The plan's code used `export let locale = $state<SupportedLocale>(...)` and then reassigned `locale = next` in callback and setLocale functions.
- **Fix:** Changed to class-based pattern: `class LocaleState { current = $state<SupportedLocale>(...) }; export const locale = new LocaleState()`. Assignments now mutate the class property (`locale.current = next`) rather than reassigning the exported binding.
- **Files modified:** `src/lib/i18n/locale.svelte.ts`
- **Commit:** `c21ff2d`

**2. [Rule 1 - Bug] No onSetLocale callback in Paraglide v2.16.0 runtime**
- **Found during:** Task 1 runtime inspection
- **Issue:** The plan expected `onSetLocale` or `onSetLanguageTag` to be exported by `src/lib/paraglide/runtime.js`. Neither is exported. The callback-based bridge pattern described in the research is not available in this runtime version.
- **Fix:** Removed callback bridge. Instead, `locale.current = next` is assigned directly in both `setLocale()` and `initLocale()` after calling `paraglideSetLocale(next, { reload: false })`. Functionally equivalent — every locale change path updates `$state`.
- **Files modified:** `src/lib/i18n/locale.svelte.ts`
- **Commit:** `c21ff2d`
