## Decisions Made
- `paraglideVitePlugin` must be imported from `@inlang/paraglide-js` (main package), not `@inlang/paraglide-js/vite` — the sub-path specifier `/vite` is absent from the package's exports map in v2.16.0. Both `import from "@inlang/paraglide-js/vite"` and `import from "@inlang/paraglide-js/dist/bundler-plugins/vite.js"` fail with esbuild `externalize-deps` because it enforces the exports map. The main index re-exports `paraglideVitePlugin` from `dist/bundler-plugins/index.js`, so the correct import is from the root package.
