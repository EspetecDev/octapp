## Component Inventory

New components and surface patterns needed for Phase 2. All must follow established Phase 1 patterns (100dvh, dark bg, mobile-first, touch-action manipulation).

### /admin/setup — Pre-Event Setup Page

**Layout:** Single scrollable column, 100dvh minimum, bg-bg, px-md (16px).

**Page header:** "Configure Game" heading (Heading/24px/bold) + "Back to Dashboard" text link (Label/14px, text-secondary). Back link must preserve `?token=` query param.

**Chapter accordion block:**
- Container: bg-surface, rounded-xl, border border-border, p-lg (24px).
- Collapsed state: chapter number + name input preview visible at all times (no fully collapsed chapters — D-02: admin sees everything at once). All chapter blocks start expanded by default.
- Chapter name: text input, full width, Heading size (24px) inline editable, placeholder "Chapter name (e.g. The Bar)".
- Minigame type selector: 3-option button group ("Trivia" / "Sensor" / "Memory"), selected state uses border-accent-admin. Each option: min-height 44px, px-md.
- Trivia question sub-block: nested inside chapter panel, shown only when minigameType === "trivia". Each question entry: question text input (full width) + correct answer input + 3 wrong option inputs arranged 1+3 grid. "Add question" button below the last question (Label/14px, text-secondary, left-aligned). "Remove" per-question: small text button (14px, text-destructive), right-aligned.
- Scavenger clue: textarea, 3 rows, full width, placeholder "Riddle or clue directing the groom where to go".
- Scavenger hint: single-line input, optional, placeholder "Optional hint (costs points to reveal)".
- Reward: textarea, 2 rows, full width, placeholder "What is unlocked (dare, location, embarrassing content)".
- "Remove Chapter" button: text-destructive (14px), bottom-right of chapter block. No confirmation dialog — destructive but low-stakes at setup time before event.

**Power-up catalog section:**
- Section heading: "Power-ups & Sabotages" (Heading/24px/bold), below all chapter blocks.
- Each entry: bg-surface card, border border-border, p-md (16px). Inline grid: name input (body/16px) | description input (body/16px) | token cost input (number, 60px wide) | effect type select ("timer_add" / "scramble_options" / "distraction"). "Remove" text button (Label/14px, text-destructive), right of entry.
- "Add Power-up" button: full-width, min-height 44px, border border-accent-admin, text-accent-admin (label/14px). Bottom of catalog section.

**Save button:**
- Full-width, min-height 48px, bg-accent-admin, text-text-primary, font-bold, rounded-xl.
- Label: "Save Setup". 
- On success flash: button text changes to "Saved" + success color for 1.5s, then reverts.
- Sticky at bottom of scrollable view: `position: sticky; bottom: 0; pb-lg;` — always visible without scrolling to bottom.
- Disabled state (opacity-50, pointer-events-none) when any required field is empty (chapter name, at least 1 trivia question if type=trivia, scavenger clue, reward).

**Validation errors:**
- Inline, below each invalid field. Text: Label/14px, text-destructive. No modal dialogs.
- Error copy: "[Field name] is required." for empty required fields. "Add at least one trivia question." for empty trivia pool.

**"Add Chapter" button:**
- Below last chapter block, above power-up section. Full-width, min-height 44px, border border-border dashed, text-text-secondary (label/14px), rounded-xl.
- Label: "+ Add Chapter". Disabled and grayed (opacity-40) when chapter count = 5 (D-04 max).

### /admin — Live Dashboard (Expanded)

Extends existing Zone 1 (session code) + Zone 2 (player list). Adds Zone 3 (chapter control) and Zone 4 (scores) per D-08.

**Zone layout:** Vertical scroll column. Zone 1 keeps its 40dvh height. Zones 2–4 share the remaining scroll area.

**Zone 3 — Chapter Control:**
- Section label: "Game Progress" (Label/14px, text-secondary).
- During lobby (no chapter unlocked): single "Unlock Chapter 1" button. Full-width, min-height 48px, bg-accent-admin, text-text-primary, font-bold, rounded-xl.
- During active game: "Chapter N of M — active" status line (Body/16px, text-primary) + "Unlock Chapter N+1" button (same styling as lobby button). Button hidden (not just disabled) when all chapters are complete.
- "Configure Game" link: renders as a text button (Label/14px, text-accent-admin) above the unlock button. Only visible during lobby state. Hidden after first chapter is unlocked (D-03). Navigates to `/admin/setup?token={token}`.

**Zone 4 — Scores:**
- Section label: "Scores" (Label/14px, text-secondary).
- Per-player row: bg-surface, border border-border, rounded-lg, h-12, px-md. Player name (Body/16px, text-primary, flex-1) + score value (Heading/24px, font-bold, text-text-primary, right-aligned).
- Score 0 shown as "0" — not hidden (D-07 says display zeros).
- Groom player row uses accent-groom left border (4px) to visually distinguish.
- Empty state (no players): "No players connected yet." (Body/16px, text-secondary, centered).

### Recap Card Overlay (all player routes: /groom, /party)

Visual treatment: full-screen overlay per CONTEXT.md `<specifics>` — "theatrical: full-screen dark background, large chapter number, chapter name in the accent color."

**Structure:**
- Outer: `position: fixed; inset: 0; z-index: 50;` bg-bg at 95% opacity (slight transparency to show context beneath).
- Inner: centered column (flexbox), gap-xl (32px), px-3xl (64px) top/bottom padding.
- Chapter number: "CHAPTER" (Label/14px, text-secondary, letter-spacing 0.2em, uppercase) + number digit (Display/40px, font-bold, text-text-primary).
- Chapter name: Heading/24px, font-bold, color: accent-groom (#f59e0b). This is the theatrical highlight.
- Progress indicator: "N of M" sub-label below chapter name (Label/14px, text-secondary). Only shown after Chapter 1 unlock.
- No dismiss button — auto-dismiss at 3 seconds (D-11). No interaction needed.
- Entry animation: fade-in + slight scale (0.95 → 1.0) over duration-normal (200ms).
- Exit animation: fade-out over duration-normal (200ms) — CSS class toggle (`.visible`) pattern, matching Phase 1 ReconnectingOverlay pattern.

## Interaction Contracts

### Form Auto-Save vs. Explicit Save

Setup form uses explicit save only (D-01/D-02/CONTEXT.md pattern). The "Save Setup" button sends `SAVE_SETUP` to the server. There is no auto-save, no draft state, and no beforeunload warning. Rationale: server memory is the source of truth; if admin refreshes, the last saved state is restored via `STATE_SYNC`.

### Chapter Unlock — No Confirmation

One-tap unlock per D-09. No modal, no "Are you sure?" prompt. The button only appears when appropriate context exists (chapters configured + previous chapter complete / lobby for Chapter 1). Accidental taps are guarded by context, not confirmation dialogs.

### Form Field Interaction States

| State | Visual Treatment |
|-------|-----------------|
| Default | border-border (#2d2d2f), bg-surface (#1c1c1e) |
| Focus | outline: 2px solid #f9fafb, outline-offset 2px (global :focus-visible rule) |
| Error (invalid) | border-destructive (#ef4444), error text below field |
| Disabled | opacity-50, pointer-events-none |

### Minigame Type Selector

3-button toggle group (not a `<select>`). Fully keyboard and touch navigable. Selected state: border-2 border-accent-admin text-text-primary bg-surface. Unselected: border border-border text-text-secondary. Min-height 44px each (MOBX-02).

### Recap Card Dismissal

No user action required. `setTimeout(3000)` triggers `showRecap = false`. CSS class toggle (`.visible` / no class) drives the fade-out animation so the 200ms exit animation completes before DOM removal — matching Phase 1 LandscapeOverlay / ReconnectingOverlay pattern.
