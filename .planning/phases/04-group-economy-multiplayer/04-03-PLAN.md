**RadialCountdown.svelte changes:**

Replace the props declaration to promote `remaining` to a `$bindable()` prop:

```svelte
let { duration, onExpire, remaining = $bindable(duration) }:
  { duration: number; onExpire: () => void; remaining?: number } = $props();
```

Remove the line: `let remaining = $state(duration);`

The interval logic stays unchanged — it still decrements `remaining` each second and calls `onExpire` at 0. Because `remaining` is now `$bindable`, when a parent binds it (`bind:remaining={timerRemaining}`), the parent owns the value and both parent and child can read/write it. When unbound (SensorMinigame, MemoryMinigame), `$bindable(duration)` defaults to `duration` and the countdown self-manages exactly as before.

The `dashOffset` and `strokeColor` `$derived` blocks remain unchanged — they read from `remaining` which is still reactive.

7. Add $effect to handle lastEffect (Pitfall 5: keep gameplay effects here, announcement overlay in groom page ONLY):
```typescript
$effect(() => {
  const effect = $lastEffect;
  if (!effect) return;

  if (effect.effectType === "timer_add" || effect.effectType === "timer_reduce") {
    const delta = effect.delta ?? (effect.effectType === "timer_add" ? 5 : -5);
    timerRemaining = Math.max(0, timerRemaining + delta); // Pitfall 2 clamp
    timerFlashDelta = delta;
    showTimerFlash = true;
    setTimeout(() => { showTimerFlash = false; }, 1000);
  }

  if (effect.effectType === "scramble_options") {
    shuffleSeed += 1; // Pitfall 7: force $derived re-evaluation
  }

  if (effect.effectType === "distraction") {
    distractionKey += 1; // new key = destroy+remount emoji storm
    showDistraction = true;
    setTimeout(() => { showDistraction = false; }, 4000);
  }
});
```
