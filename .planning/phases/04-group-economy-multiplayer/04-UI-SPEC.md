**Accent reserved for:**
- Group accent (`#ef4444`): earn button background, token balance pill, sabotage-type announcement overlay tint (timer_reduce / scramble_options / distraction), groom "-5s" flash, "-5s" overlay text
- Groom accent (`#f59e0b`): power-up-type announcement overlay tint (timer_add), "+5s" flash on groom timer, ✨ prefix color on announcement overlay

**Overlay tint formula (from D-09):**
- Sabotage overlay: `rgba(239, 68, 68, 0.85)` — red at 85% opacity over `#0f0f0f` base
- Power-up overlay: `rgba(245, 158, 11, 0.85)` — gold at 85% opacity over `#0f0f0f` base

**0-token state:** Shop items with cost > current balance are visually dimmed (`opacity: 0.4`) with the "Spend" button disabled (`disabled` attribute + `cursor: not-allowed`). Items are NOT hidden — the player can see what is unavailable and what they need to save for. No "Not enough tokens" text per item — the balance shown prominently above makes the reason self-evident.

## Component Inventory

### New Components (Phase 4)

#### 1. GroupEconomyScreen (party page active-challenge state)

Single-screen layout, full-height (`min-h-[100dvh]`), `flex flex-col`, no scroll on the outer container. Three stacked zones:

**Zone 1 — Groom Progress Bar (top, fixed height 48px)**
- Thin pill/strip spanning full width with `bg-surface border-b border-border`
- Content: chapter name (label, 14px, text-secondary) on left + live countdown value (heading, 24px, bold, text-primary) on right
- Timer value reads from `gameState.chapters[activeChapterIndex].timerRemaining` (new field on GameState)
- If no active minigame (timerRemaining is null): show "Waiting for groom..." in text-secondary

**Zone 2 — Earn Area (middle, flex-1)**
- Centered vertically within its flex-1 container
- Token balance display: `"💰 {N} tokens"` — display size (40px), bold, text-primary, centered
- Earned counter: `"{earned} / 5 earned this challenge"` — label size (14px), text-secondary, centered, below balance
- Earn button: full-width (max-width 280px), 80px height, `bg-accent-group rounded-2xl`, bold 24px label
  - Label: **"TAP TO EARN"** (locked wording — clear, imperative, impossible to misread at a party)
  - Active/unlocked state (earned < 5): normal opacity, `active:scale-95 transition-transform duration-100` for tactile press response
  - Cap reached state (earned >= 5): `opacity-50`, button is disabled, label changes to **"MAX EARNED"**
  - On each successful tap: `navigator.vibrate(50)` (50ms short pulse — distinct from minigame win 200ms)
  - On each successful tap: brief `bg-success` flash for 200ms then back to `bg-accent-group` (CSS transition)
- Phase name label (above earn button, optional): chapter name at 14px text-secondary + letter-spacing, uppercase. Example: "CHAPTER 2 — TRIVIA CHALLENGE". Resolves Claude's discretion "whether to show chapters counter" — YES, shown as a small label above the earn button.

**Zone 3 — Shop (bottom, fixed max-height 40vh, overflow-y auto)**
- Header: "GROUP SHOP" — label size (14px), text-secondary, uppercase, letter-spacing 0.15em
- Each shop item: card row (`bg-surface border border-border rounded-lg px-4 py-3`, min 64px height)
  - Left: item name (16px bold, text-primary) + description (14px, text-secondary, max 2 lines)
  - Right: cost badge (`{N} tokens`, 14px, rounded pill, bg `#1c1c1e` border `#2d2d2f`) + "Spend" button (48px wide, 36px height, `bg-accent-group rounded-lg`, disabled state as above)
- Items filtered by `chapter.minigameType` (client-side, D-16/D-17)
- Items sorted: timer_add first (positive), then timer_reduce and distraction (sabotages)
- Scroll container: `-webkit-overflow-scrolling: touch; overscroll-behavior: contain`

#### 2. SocialWaitingScreen (party page lobby/scavenger/reward state)

Replaces the existing static waiting content in `src/routes/party/+page.svelte`. Recap and reward overlays continue to render on top (existing behavior unchanged).

Layout: `flex flex-col gap-lg px-md py-lg`, scrollable main area.

**Player balances list:**
- Header: "TOKEN BALANCES" — label size (14px), text-secondary, uppercase, letter-spacing 0.15em
- Each row: player name (16px bold, text-primary) + token balance pill (`"💰 {N}"`, 14px, `bg-surface rounded-full px-sm py-xs border border-border`)
- Rows in a `ul` with `border border-border rounded-lg bg-surface overflow-hidden`
- Each `li`: `px-md py-sm border-b border-border last:border-0`, min 44px height

**Recent actions feed:**
- Header: "RECENT ACTIONS" — label size (14px), text-secondary, uppercase, letter-spacing 0.15em
- Each entry: `"{playerName} used {powerUpName}"` — body 16px text-primary + timestamp metadata (14px text-secondary), two-line row
- Sabotage entries: `#ef4444` left border strip (4px solid) for visual distinction
- Power-up entries: `#f59e0b` left border strip (4px solid)
- Max 10 entries displayed (newest first), from `gameState.recentActions`
- Empty state: "No actions yet. Be the first to use a power-up!" (14px text-secondary, centered, padded)

#### 3. AnnouncementOverlay (all clients — groom page, party page, admin page)

Full-screen overlay, `position: fixed; inset: 0; z-index: 100` (above all existing overlays which use z-index 50).

CSS pattern mirrors existing recap-overlay:
```css
.announcement-overlay {
  position: fixed;
  inset: 0;
  z-index: 100;
  display: flex;
  flex-direction: column;
  align-items: center;
  justify-content: center;
  opacity: 0;
  pointer-events: none;
  transition: opacity 200ms ease; /* --duration-normal */
}
.announcement-overlay.visible {
  opacity: 1;
  pointer-events: none; /* always none — auto-dismisses, no user action needed */
}
```

Background: sabotage = `rgba(239, 68, 68, 0.85)` over `#0f0f0f`; power-up = `rgba(245, 158, 11, 0.85)` over `#0f0f0f`.

Content (centered column, gap 16px):
- Line 1: activating player's name — **64px, bold, text-primary, uppercase, letter-spacing 0.05em**
- Line 2: effect prefix + power-up name — **40px (display), bold, text-primary** — prefix is ⚡ for sabotages, ✨ for power-ups

Auto-dismisses after 2000ms. Triggered by `EFFECT_ACTIVATED` message received on all clients.

Simultaneous sabotages: each new `EFFECT_ACTIVATED` restarts the 2s dismiss timer (the new announcement replaces the previous one). No stacking — the last activation wins the display.

#### 4. DistractionOverlay (groom page only)

Emoji storm: floats up from the bottom of the screen. `position: fixed; inset: 0; z-index: 60; pointer-events: none`. Active for 4000ms on receipt of `EFFECT_ACTIVATED` with `effectType: "distraction"`.

**CSS animation:**
```css
@keyframes floatUp {
  from { transform: translateY(0); opacity: 0.9; }
  to   { transform: translateY(-110vh); opacity: 0; }
}
```

Emoji set (from D-12): 🍻 👑 💀 🥳 💍 🎶 — 12 emoji elements total per activation, 2 of each.

Each emoji element:
- `position: absolute; bottom: -40px`
- Random `left` value: 5–90% (assigned inline at render time)
- Random `animation-delay`: 0–1200ms
- `animation-duration`: 2800ms–3800ms (random, assigned inline)
- `font-size`: 32px
- `animation-timing-function: ease-out`

`pointer-events: none` on the overlay container — answer buttons remain tappable beneath.

Back-to-back distraction activations: overlay is torn down and re-mounted (component key reset) to restart the animation cleanly.

#### 5. TimerDeltaFlash (groom page, on RadialCountdown)

When `EFFECT_ACTIVATED` arrives with `effectType: "timer_add"` or `timer_reduce`:
- A text node `"+5s"` or `"-5s"` appears centered over the radial countdown ring
- `position: absolute; top: 50%; left: 50%; transform: translate(-50%, -50%)`
- Font: 24px bold; color: `#f59e0b` for add, `#ef4444` for reduce
- Fade-out: `opacity: 1 → 0` over 800ms starting at t=200ms (visible for 200ms before fading)
- Dismissed at t=1000ms (total display window)

## Interaction Contracts

### Earn Tap
- Touch: fires on `touchend` (not `touchstart` — avoids accidental triggers on scroll)
- Haptic: `navigator.vibrate(50)` — 50ms (shorter than minigame win 200ms; additive tapping must not feel punishing)
- Visual feedback: earn button background flashes `bg-success` (#22c55e) for 200ms then returns to `bg-accent-group`
- Cap enforcement: client increments local `earnedThisChallenge` counter; at 5, button becomes disabled. Counter resets to 0 when `activeChapterIndex` changes.
- No server round-trip for the earn tap itself — earning is tracked client-side. Only spending (SPEND_TOKEN) hits the server.

### Token Spend
- Tap "Spend" on a shop item → immediate optimistic UI: button shows loading state (`opacity-50`, disabled) for 200ms
- Server validates and broadcasts STATE_SYNC; client reads new balance from `gameState.tokenBalances[myPlayerId]`
- If server rejects (insufficient balance race condition): show brief toast "Not enough tokens" (14px, text-secondary, dismisses after 1500ms) — no modal
- Toast position: bottom of screen, `position: fixed; bottom: 24px; left: 50%; transform: translateX(-50%)`, `bg-surface border border-border rounded-lg px-md py-sm`

### Announcement Overlay Timing
- On `EFFECT_ACTIVATED` receipt: overlay immediately becomes `.visible` (opacity 0→1, 200ms ease)
- After 2000ms: remove `.visible` class (opacity 1→0, 200ms ease)
- Total visible duration: ~2200ms including fade-out
- If a second `EFFECT_ACTIVATED` arrives during the 2s window: restart the dismiss timer (setTimeout cleared and reset), update overlay content immediately

### Emoji Storm
- Triggered by `EFFECT_ACTIVATED` with `effectType: "distraction"`
- Overlay mounted with 12 emoji elements, each with randomized x/delay/duration
- Unmounted at t=4000ms
- Back-to-back: if another distraction arrives while one is active, the overlay is destroyed and re-created (Svelte `{#key distractionKey}` pattern — increment `distractionKey` on each trigger)

### Wake Lock
- Party page does NOT acquire Wake Lock (per existing scope — Wake Lock is groom-only per MOBX-03)
- Groom page: existing Wake Lock behavior unchanged; timer delta effects do not affect Wake Lock state

## Accessibility

- All interactive elements: `min-height: 44px; min-width: 44px` (MOBX-02 — already enforced in `app.css`)
- Earn button: `aria-label="Tap to earn a token"`, `aria-disabled="true"` when cap reached
- Announcement overlay: `role="status"` + `aria-live="assertive"` (matches recap card pattern — assertive because it's time-critical game info)
- Distraction overlay: `aria-hidden="true"` (decorative — groom's functional UI beneath remains accessible)
- Shop item "Spend" button: `aria-label="Spend {cost} tokens on {name}"` for screen reader context
- Social feed: `aria-label="Recent actions"` on the `ul`
- Token balances list: `aria-label="Player token balances"` on the `ul`
