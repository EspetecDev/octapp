## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, dot pulse spacing, card inner padding |
| sm | 8px | Compact element spacing, recap-content gap, option button gap |
| md | 16px | Default element spacing, screen horizontal padding |
| lg | 24px | Section padding, card grid gap |
| xl | 32px | Layout gaps, overlay content horizontal padding |
| 2xl | 48px | Major section breaks, sensor meter vertical margin |
| 3xl | 64px | Page-level spacing, overlay content vertical padding |

Exceptions:
- Touch targets: minimum 44px height/width enforced globally via `button, [role="button"]` rule in `src/app.css` (MOBX-02)
- Memory card grid: each card cell minimum 64px × 64px to accommodate emoji at readable size

Source: `src/app.css` spacing tokens (pre-populated, not re-asked).

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0f0f0f` (`--color-bg`) | Full-viewport backgrounds, overlay base |
| Secondary (30%) | `#1c1c1e` (`--color-surface`) | Card faces (memory cards), option buttons, scavenger clue card, reward card, accordion items |
| Accent (10%) | `#f59e0b` (`--color-accent-groom`) | Reserved list below |
| Destructive / Loss | `#ef4444` (`--color-destructive`) | Loss result overlay tint, timer final-10%-zone stroke, "TIME'S UP!" text |

**Accent `#f59e0b` reserved for:**
1. Win result overlay background tint (`rgba(245,158,11,0.15)`)
2. Win result heading text ("NAILED IT!" / "CORRECT!")
3. Radial countdown stroke color when `remaining > duration * 0.5` (green `#22c55e` transitions to amber at 50%, red at 25%)
4. Groom role badge pill (existing — "You are the Groom")
5. Recap card chapter name text (existing pattern)
6. Reward reveal heading text and glow effect
7. "I found it!" primary CTA button background
8. Three-dot waiting pulse (existing — `#f59e0b`)

**Additional semantic colors (non-accent):**
- `#22c55e` (`--color-success`): Radial countdown stroke when `remaining > duration * 0.5` (green phase)
- `#f59e0b` (`--color-accent-groom`): Radial countdown stroke when `remaining > duration * 0.5` (amber phase — shares token)
- `#2d2d2f` (`--color-border`): Radial countdown track circle, card back face, memory card back tile
- `#9ca3af` (`--color-text-secondary`): Hint button label, clue sub-label, "Request Hint (−10 pts)" cost label

Source: `src/app.css` color tokens + CONTEXT.md D-18 + RESEARCH.md radial timer pattern (pre-populated).

## Component Inventory

### RadialCountdown.svelte (new — shared)

Shared component used by all three minigame types.

```
Props:
  duration: number          — total seconds (15 for trivia, 30 for memory, 30 for sensor)
  onExpire: () => void      — called when remaining hits 0

Visual:
  96px × 96px SVG (w-24 h-24)
  Track circle: stroke #2d2d2f, stroke-width 8
  Progress circle: stroke-dashoffset animated via CSS transition: stroke-dashoffset 1s linear
    - remaining > 50%: stroke #22c55e
    - remaining 25–50%: stroke #f59e0b
    - remaining < 25%: stroke #ef4444
    - stroke color transition: 300ms
  Center text: remaining seconds, 24px / bold / #f9fafb
  Transform: rotate(-90deg) at transform-origin 50% 50%

Accessibility: role="timer" aria-label="{remaining} seconds remaining"
```

Source: RESEARCH.md radial timer code example + CONTEXT.md D-13/D-16.

### Win/Loss Result Overlay (reusable across all 3 minigames)

```
Overlay base: position fixed, inset 0, z-index 50
Opacity: 0 (hidden), 1 (visible) — .visible class toggle, transition: opacity 200ms ease
pointer-events: none when hidden, auto when visible

Win state background: rgba(245,158,11,0.15) — amber tint
Loss state background: rgba(239,68,68,0.15) — red tint

Content (flex-col centered gap-md):
  1. Result text — 40px / bold
     Win (trivia): "CORRECT!"
     Win (sensor/memory): "NAILED IT!"
     Loss (trivia): "WRONG!"
     Loss (sensor/memory): "TIME'S UP!"
  2. Points delta — 24px / bold
     Win: "+50 pts" in #22c55e
     Loss: "−20 pts" in #ef4444

Auto-advance: 2 seconds, then sends MINIGAME_COMPLETE and clears overlay
Haptic:
  Win: navigator.vibrate(200) — single 200ms buzz
  Loss: navigator.vibrate([100, 50, 100]) — double buzz
  Guard: "vibrate" in navigator check (iOS Safari fails silently without guard)
```

Confetti implementation: CSS keyframe animation — 24 absolutely-positioned `<div>` elements, 6px × 6px squares, randomized `left` (0–100%), `animation-delay` (0–800ms), `animation-duration` (600–1000ms), colors cycling through `#f59e0b / #22c55e / #ef4444 / #f9fafb`. Keyframe: translate from `y: 0` to `y: -60vh`, opacity 1 → 0. Win state only. No canvas, no external library.

Source: CONTEXT.md D-18/D-20, RESEARCH.md win/loss overlay pattern. Confetti per Claude's Discretion — declared here as contract.

## Screen Transition Contract

| From | To | Trigger | Visual |
|------|----|---------|--------|
| Waiting | Minigame | UNLOCK_CHAPTER — recap card auto-dismisses (3s) | Recap card overlay fades out (existing pattern), minigame screen appears |
| Minigame | Win/Loss overlay | Timer expires OR answer selected / win condition met | Overlay fades in (200ms), auto-dismisses after 2s |
| Win/Loss overlay | Scavenger | 2s auto-advance, MINIGAME_COMPLETE sent | Overlay fades out, scavenger screen renders |
| Scavenger | Reward | SCAVENGER_DONE broadcast, `chapter.scavengerDone` flips true | Reward screen renders; party page shows reward overlay |
| Reward | Waiting (next chapter) | Admin sends UNLOCK_CHAPTER (next chapter) | Recap card for new chapter, then new minigame |

No animated page transitions between screens — abrupt render swap inside `{#if}` blocks is intentional (party-game pace).

## Wake Lock Contract

| Screen | Wake Lock State |
|--------|----------------|
| Waiting | Released |
